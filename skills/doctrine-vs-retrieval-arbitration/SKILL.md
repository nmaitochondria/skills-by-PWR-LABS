# Arbitrate doctrine vs ToolUniverse

## What this unlocks

Separates **what the product promises** (doctrine, sparse evidence posture) from **how facts are retrieved** (ToolUniverse / routers)—so clever retrieval never silently replaces a governance decision.

## Origin in the [PWR] ecosystem

**[synthesis.]** tension surface: scientific routing vs product commitments; default bias toward doctrine when shipping behavior, ToolUniverse when grounding facts.

## Broader applications

- Any RAG-heavy product needing an explicit split between policy and retrieval.
- Compliance narratives where evidence grading must not collapse into confident prose.
- Internal training for PMs and scientists collaborating on the same repo.

## When to use

- A question could be answered by **ToolUniverse** tools **or** by **product doctrine / handoff** truth.
- Risk that the **router** (`tooluniverse`) steers work that should be a **connector → doctrine** decision (schema, evidence policy, human-gated merges).

## Two layers (keep separate)

| Layer | Answers |
| --- | --- |
| **Product doctrine** | What **[synthesis.]** promises operators: evidence vs judgment, uncertainty typing, destructive vs additive schema policy, roadmap honesty. |
| **ToolUniverse** | How to **retrieve and compute** over public datasets (2300+ tools) with domain-specific **`tooluniverse-*`** skills. |

## Default bias

1. If the user is deciding **product behavior, contracts, or what ships** → **`docs/PRODUCT_DOCTRINE.md`**, **`docs/CONNECTOR_DIALOGUE_THREAD.md`**, **`docs/AGENT_HANDOFF.md`** win.
2. If the user needs **grounded scientific facts** from databases → route via **`tooluniverse`** / specialized **`tooluniverse-*`** skills.
3. If both apply → **state the split explicitly** (“ToolUniverse answers X; doctrine commits Y”) and avoid mixing them in one unlabeled paragraph.

## Anti-patterns

- Letting a **clever retrieval** replace a **changelogged** architectural decision.
- Hiding **sparse evidence** behind confident prose when doctrine requires typed coverage notes.
